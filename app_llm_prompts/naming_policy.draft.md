# 命名ポリシー（ドラフト）

これはランタイム文書解析 LLM と命名ロジックのドラフトポリシーです。

Claude Code の指示ではありません。

## 基本的な出力ファイル名

以下の概念的なフォーマットを使用します：

```text
<文書日付>_<年分・年度分・期間補足>_<書類タイトル>[_発行元].pdf
```

該当しないコンポーネントは省略します。

ファイル名に `対象` という言葉を使わないでください。

## 日付

識別できる場合は文書の発行日、作成日、または交付日を使用します。

文書の日付が見つからない場合は、元のスキャナーファイル名から導出したスキャンタイムスタンプを使用します。

ソース文書が元号表記で日付を記載している場合は、元号表記を括弧内に含めます。

例：

```text
20211001(R3)_住宅取得資金に係る借入金の年末残高等証明書.pdf
```

文書が西暦のみで日付を記載している場合は、元号表記を強制しません。

## 年分文書

確定申告書など暦年に基づく文書の場合は以下を使用します：

```text
<YYYY年分>
```

例：

```text
20260310(R8)_2025年分_確定申告書第一表.pdf
```

## 年度文書

年度ベースの文書の場合は以下を使用します：

```text
<YYYY年度分(YYYYMM-YYYYMM)>
```

例：

```text
20260415(R8)_2025年度分(202504-202603)_医療費通知.pdf
```

## 発行者・銀行名

ファイルを区別するために実質的に役立つ場合は発行者を含めます。

発行者が信頼性高く識別できる場合は、住宅ローンや融資関連の証明書に発行者を含めます。

例：

```text
20211001(R3)_住宅取得資金に係る借入金の年末残高等証明書_三井住友銀行.pdf
```

タイトルだけで十分でノイズになる場合は発行者を含めません。

多くの官公庁文書では、タイトルと年分・年度分情報だけで十分な場合があります。

## タイトル

公式な文書タイトルを使用します。

長い説明文、住所、受取人名、本文テキストをタイトルに含めないでください。

以下のような簡潔なタイトルを優先します：

```text
住宅取得資金に係る借入金の年末残高等証明書
確定申告書第一表
医療費通知
特別区民税・都民税納税通知書
```

## 要確認

文書のタイトル、日付、または必要な発行者が不確かな場合、アプリケーションは最終的な名前を生成する代わりにファイルをレビュー対象としてマークすることがあります。
