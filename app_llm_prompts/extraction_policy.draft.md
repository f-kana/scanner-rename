# 抽出ポリシー（ドラフト）

これはランタイム LLM 抽出のドラフトポリシーです。

実装では、ランタイム LLM に対して散文ではなく構造化データを返すよう指示します。

## 必須抽出フィールド

アプリケーションは以下の抽出を試みます：

```json
{
  "document_date": {
    "value": "YYYY-MM-DD",
    "source_uses_japanese_era": true,
    "japanese_era": "R8",
    "evidence": "...",
    "confidence": 0.0
  },
  "title": {
    "value": "...",
    "evidence": "...",
    "confidence": 0.0
  },
  "issuer": {
    "value": "...",
    "evidence": "...",
    "confidence": 0.0
  },
  "document_type": {
    "value": "...",
    "confidence": 0.0
  },
  "year_part": {
    "value": "2025年分",
    "evidence": "...",
    "confidence": 0.0
  },
  "fiscal_year_part": {
    "value": "2025年度分",
    "period": "202504-202603",
    "evidence": "...",
    "confidence": 0.0
  },
  "reason": "短い説明"
}
```

正確なスキーマは実装中に変更される可能性があります。

## 日付の優先順位

複数の日付が含まれる場合、以下の優先順位で選択します：

1. 発行日
2. 作成日
3. 交付日
4. 領収書の場合は領収日
5. 請求書の場合は請求日
6. その他の目立つ文書ヘッダーの日付
7. 信頼できる日付が見つからない場合は LLM 外でスキャンタイムスタンプにフォールバック

## 元号

ソース文書で元号表記が使われている場合は、元号のシグナルを保持します。

例：

```text
令和8年3月10日 -> 2026-03-10 with japanese_era=R8
令和3年10月1日 -> 2021-10-01 with japanese_era=R3
```

## 年分・年度分

確定申告書などの暦年文書の場合は `YYYY年分` を抽出します。

年度文書の場合は可能であれば以下の両方を抽出します：

```text
YYYY年度分
YYYYMM-YYYYMM
```

## エビデンス

主要なフィールドのエビデンスの抜粋を返します。

エビデンスは短くします。

OCR テキスト全体を返さないでください。

## 信頼度

信頼度の値を使ってアプリケーションの判断を支援します：

- 通常のリネーム
- 要確認
- エラーは通常の不確実性の範囲外の技術的障害のみ
