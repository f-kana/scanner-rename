# PJ固有の要求で自作を推奨する

## Prompt

classifying-harnessでカテゴリ4（Skill新規）に分類済み。
「日本語の社内レビュープロセスに合わせたコードレビューSkill」が欲しい。
社内の承認フロー、日本語でのレビューコメント規約、部署固有のチェック項目がある。
既存の公開レビューSkillを調査してほしい。

## Expected behavior

- 候補検索を実施し、探索元を明示する
- 既存候補のFitをLow〜Mediumと評価する（PJ固有の要件が多いため）
- 「参考にして自作」または「完全自作」を推奨する
- 自作する場合の骨子（name、主な責務、対象外）を提示する
- 参考にできる部分があれば言及する

## Failure behavior to avoid

- 探索せずにいきなり「自作」と結論づける
- 汎用レビューSkillをFit Highと過大評価する
- 自作の骨子を提示しない

## Pass criteria

- 探索範囲が明示されている
- Fitの評価理由にPJ固有性への言及がある
- 自作推奨の理由が説明されている
- 自作の骨子が出力されている

## Negative case

「TypeScriptプロジェクトのlint設定Skill」の場合。
公式/有力な候補が多数存在するため、自作ではなく既存採用を検討すべき。
